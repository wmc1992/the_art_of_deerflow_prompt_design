# 如何使用本书

## 本书的四种读法

本书围绕同一份源码，从四个层次展开分析，不同背景、不同目标的读者可以选择最适合自己的入口。

### 读法一：从头到尾（推荐首次阅读）

按照章节顺序阅读，每章约 30-50 分钟。适合希望系统建立 Prompt 工程认知框架的读者。

**阅读路径**：序言 → 第 1-3 章（基础）→ 第 4-7 章（主系统 Prompt）→ 第 8-10 章（子系统）→ 第 11-12 章（设计模式与迁移）→ 附录 A/B（参考）

### 读法二：从附录出发（快速上手）

直接阅读附录 A（所有 Prompt 英文原文），遇到不理解的设计选择时，按照附录 B 的中文译注或正文章节来索引解释。

**适合**：已有 Prompt 工程经验、希望快速获取参考素材的工程师

### 读法三：按设计模式阅读（提炼原则）

先读第 2 章（核心原则）和第 11 章（七大设计模式），建立模式词汇表，再根据感兴趣的模式回溯到具体的 Prompt 章节。

**适合**：需要将经验迁移到自己项目的架构师或技术负责人

### 读法四：按子系统阅读（专项深入）

如果你只关心某个具体方向，可以按子系统跳读：

| 关注方向 | 推荐章节 |
|---------|---------|
| 主 Agent 系统 Prompt | 第 4-7 章 + 附录 A 的 P-01 |
| 多 Agent 编排 | 第 8 章 + 附录 A 的 P-02、P-17、P-18、P-19、P-20、P-22 |
| 记忆与长期对话 | 第 9 章 + 附录 A 的 P-10、P-11 |
| 工具设计 | 第 7 章 + 附录 A 的 P-21 至 P-25 |
| 安全与防护 | 第 10 章 + 附录 A 的 P-13 至 P-16 |
| 技能系统 | 第 6 章 + 附录 C（22 个技能原文）|

## 每章的结构

正文各章遵循统一的内部结构：

1. **本章导读**：本章分析哪些 Prompt，解决什么问题
2. **原文展示**：完整引用相关 Prompt 的英文原文（与附录 A 一致）
3. **逐段解构**：每个设计选择的动机、效果和取舍
4. **设计模式**：从具体案例提炼的可复用模式（有编号，见第 11 章汇总）
5. **举一反三**：如果你的场景不同，如何调整这些设计

动态生成的 Prompt（如 P-02）会同时展示**模板原文**（含 `{placeholder}` 占位符）和一个**渲染示例**（默认配置下的实际输出），帮助你理解运行时的真实面貌。

---

## 关于 Prompt 编号

本书使用两套编号体系：

- **P-XX（Prompt）**：Python 源码中的 Prompt，P-01 至 P-25，来自 `backend/packages/harness/` 下的各模块
- **S-XX（Skill）**：技能文件，S-01 至 S-22，来自 `skills/public/` 目录下的各 SKILL.md

这两套编号是本书自行定义的，原始代码中没有对应命名。所有编号都可以在目录页（`index.md`）中查到完整索引。

---

## 关于代码引用

本书引用的所有英文 Prompt 都与 DeerFlow v2.0.0-rc1 源码完全一致。为了在 Markdown 中正确展示内嵌代码块（某些 Prompt 本身包含 ` ``` ` 代码示例），部分外层代码围栏使用 4 个反引号（` ```` `）以区分内外层级。

正文分析段落中引用 Prompt 片段时使用的是 `>` 引用块或行内代码格式，不会对原文作任何修改。

## 阅读本书所需的背景知识

本书假设读者具备以下背景：

**必要背景**：
- 了解大语言模型（GPT/Claude/Gemini）的基本工作方式：token、上下文窗口、对话轮次
- 有过 Prompt 工程的实践经验（写过 system prompt，调过模型行为）
- 能阅读 Python 代码（理解变量、字符串、函数调用即可，不需要深入的 Python 知识）

**有帮助但非必须**：
- 了解 LangChain/LangGraph 的基本概念（Graph、Node、Edge、Message）
- 了解 Function Calling / Tool Use 的工作原理
- 了解多 Agent 系统的基本概念

如果你对某些背景概念不熟悉，第 1 章的"系统架构简介"部分会提供足够的上下文让你继续阅读。
